# Nexus-Node CI 重构 — 工作任务安排

> 日期：2026-03-31
> 前置文档：`01-CI审计报告.md`、`02-CI流程设计方案.md`

---

## 总体原则

- **增量迁移**：不一次性替换所有 workflow，而是分阶段逐步切换
- **旧新并行**：新 workflow 先 `workflow_dispatch` 手动触发验证，确认无误后再切到自动触发
- **回退机制**：旧 workflow 文件保留为 `ci-legacy.yml`，出问题可立即切回
- **每步可验**：每个任务完成后均可独立验证

## 阶段一：基础设施准备（预计 1-2 天）

### Task 1.1: 验证 cargo nextest archive 可行性

**目标**：确认 nextest archive 能覆盖当前所有测试场景

```bash
# 在本地执行验证
cargo install cargo-nextest

# 创建默认测试存档
cargo nextest archive --workspace --archive-file /tmp/nexus-test-archive.tar.zst

# 用存档运行全量测试
cargo nextest run --archive-file /tmp/nexus-test-archive.tar.zst

# 验证指定过滤表达式可用
cargo nextest run --archive-file /tmp/nexus-test-archive.tar.zst \
  -E 'test(forged_vote_should_be_rejected)'

# 创建 VM 特性存档
cargo nextest archive \
  -p nexus-execution -p nexus-wallet -p nexus-test-utils \
  --features move-vm \
  --archive-file /tmp/nexus-vm-test-archive.tar.zst

# 验证 --ignored 测试（crypto KAT）
cargo nextest run --archive-file /tmp/nexus-test-archive.tar.zst \
  -p nexus-crypto -- --ignored --test-threads=1
```

**验证标准**：

- [ ] 默认存档测试数量 ≥ 当前 `cargo nextest run` 数量
- [ ] 所有 ci.yml 中的 `-E 'test(...)'` 过滤表达式可正常工作
- [ ] `--ignored` 测试可通过存档运行
- [ ] VM 特性存档可正常运行 Move VM 测试
- [ ] 存档文件大小合理（< 500MB）

**已知限制**：

- doc-tests 不支持 nextest，需要在 build Job 中用 `cargo test --doc` 运行
- 如果有些测试依赖运行时工作目录结构，需要在 nextest run 时提供 `--workspace-remap`

### Task 1.2: 验证 Docker image artifact 方案

**目标**：确认 docker save/load + zstd 压缩方案可行

```bash
# 构建 release 二进制
cargo build --release -p nexus-node -p nexus-keygen -p nexus-genesis -p nexus-wallet

# 构建 Docker 镜像
docker build --target runtime-prebuilt -t nexus-node:test .

# 保存并压缩
docker save nexus-node:test | zstd -1 > /tmp/nexus-image.tar.zst
ls -lh /tmp/nexus-image.tar.zst  # 检查大小

# 加载并验证
docker rmi nexus-node:test
zstd -d /tmp/nexus-image.tar.zst | docker load
docker run --rm nexus-node:test --version
```

**验证标准**：

- [ ] 压缩镜像 < 200MB（适合 GitHub artifact 上传）
- [ ] load 后镜像可正常启动
- [ ] 健康检查端点可访问

### Task 1.3: 验证 devnet 可使用外部二进制

**目标**：确认 setup-devnet.sh 和 generate-compose.sh 可以使用从 artifact 下载的二进制

```bash
# 模拟 artifact 下载目录结构
mkdir -p /tmp/nexus-bin
cp target/release/nexus-{node,keygen,genesis,wallet} /tmp/nexus-bin/
export PATH="/tmp/nexus-bin:$PATH"

# 测试 devnet 引导
./scripts/setup-devnet.sh -n 4 -o /tmp/devnet-test -f
./scripts/generate-compose.sh -n 4 -o /tmp/docker-compose-test.yml -d /tmp/devnet-test
```

**验证标准**：

- [ ] setup-devnet.sh 可以在 PATH 中找到 nexus-keygen/nexus-genesis
- [ ] 生成的 docker-compose.yml 使用正确的镜像名称

## 阶段二：编写新 Workflow（预计 2-3 天）

### Task 2.1: 创建 `ci-v2.yml`（手动触发版本）

创建新的工作流文件，初始配置为 `workflow_dispatch` 手动触发，方便验证。

文件：`.github/workflows/ci-v2.yml`

**Job 列表**：

| Job | 依赖 | 核心步骤 | 预估耗时 |
| ----- | ------ | --------- | --------- |
| `preflight` | — | fmt-check, security-audit, config-drift, machete | 3 min |
| `build` | preflight | clippy, release build, nextest archive ×2, docker image, upload artifacts | 20 min |
| `core-tests` | build | nextest run (全量), doc-tests, crypto-kat | 12 min |
| `safety-tests` | build | negative Phase A-F, recovery, cold-restart | 10 min |
| `integration-tests` | build | readiness, epoch, proof-surface | 8 min |
| `vm-tests` | build | Move VM nextest run | 5 min |
| `formal-verify` | build | FV diff, proptest, stake-quorum | 8 min |
| `capacity-gas` | build | capacity-curves, gas-calibration | 5 min |
| `smoke-tests` | build, core-tests, safety-tests, vm-tests | devnet 4-node + contract smoke | 12 min |
| `coverage` | build | llvm-cov (独立编译) | 20 min |
| `release-go-nogo` | all above | go/no-go script | 2 min |

### Task 2.2: 测试 Job 模板标准化

编写每个测试 Job 的精确配置，包括：

- nextest archive 下载
- 过滤表达式验证（确保覆盖所有原有测试）
- 超时设置

**映射关系清单**（原 Job → 新 Job 中的测试步骤）：

```text
原 test            → core-tests: nextest run --profile ci
原 test-vm         → vm-tests: nextest run --archive-file nexus-vm-test-archive.tar.zst
原 correctness-neg → safety-tests: nextest run -E 'test(...Phase A-F...)'
原 startup-ready   → integration-tests: nextest run -E 'test(readiness_|chaos_readiness_|lifecycle_|node_e2e_)'
原 epoch-reconfig  → integration-tests: nextest run -E 'test(epoch_|governance_recovery_)'
原 stake-quorum    → formal-verify: nextest run -E 'test(j8_|j9_|k1_|stake_weighted_|economic_foundation_)'
原 proof-surface   → integration-tests: nextest run -E 'test(proof_|commitment_|snapshot_signing_|epoch_e2e_consistency_)'
原 recovery        → safety-tests: nextest run -E 'test(recovery_tests)'
原 cold-restart    → safety-tests: nextest run -E 'test(cold_restart_|persistence_)'
原 capacity-curves → capacity-gas: nextest run -E 'test(config_defaults_|tier_hierarchy_|capacity_curve_|cross_class_|quota_manager_|multi_ip_)'
原 gas-calibration → capacity-gas: nextest run -E 'test(gas_calibration)'
原 formal-verify   → formal-verify: nextest run -E 'test(fv_diff_|fv_proptest)'
原 crypto-kat      → core-tests: nextest run -p nexus-crypto -- --ignored
原 coverage        → coverage (独立 Job，需要编译)
原 config-drift    → preflight (脚本检查，无需编译)
原 check           → build (clippy 隐含 check)
```

### Task 2.3: 冒烟测试 Job 整合

将 `devnet-smoke.yml` 的逻辑和 `ci.yml` 中的 `move-vm-smoke` 合并为一个 `smoke-tests` Job。

**关键差异处理**：

| 原 workflow | 节点数 | 额外步骤 | 新 Job 中的处理 |
| ------------- | ------- | --------- | ---------------- |
| devnet-smoke.yml | 4 | 基础健康检查 | smoke-tests 步骤 1-6 |
| move-vm-smoke | 7 | 合约冒烟测试 | smoke-tests 步骤 7（使用 4 节点） |

**决策**：统一使用 4 节点（足以验证共识和合约部署），节省资源。

### Task 3.2: 切换触发器

确认新 workflow 稳定后：

1. 将 `ci-v2.yml` 的触发器从 `workflow_dispatch` 改为 `push/PR → main`
2. 将 `ci.yml` 重命名为 `ci-legacy.yml`，触发器改为 `workflow_dispatch`（保留回退能力）
3. 删除 `devnet-smoke.yml`（功能已合入 smoke-tests Job）

## 附录：快速参考命令

### 本地模拟新 CI 流程

```bash
# Phase 0: Pre-flight
cargo fmt --all -- --check
cargo audit --ignore RUSTSEC-2025-0009 --ignore RUSTSEC-2026-0049 --ignore RUSTSEC-2026-0009
cargo deny check licenses bans sources
zsh scripts/config-doc-drift-check.sh

# Phase 1: Build
cargo clippy --workspace --all-targets -- -D warnings
cargo build --release -p nexus-node -p nexus-keygen -p nexus-genesis -p nexus-wallet
cargo nextest archive --workspace --archive-file /tmp/nexus-test-archive.tar.zst
cargo test --doc
docker build --target runtime-prebuilt -t nexus-node:local .

# Phase 2: Tests (from archive)
cargo nextest run --archive-file /tmp/nexus-test-archive.tar.zst --profile ci

# Phase 3: Smoke
make ci-local-smoke

# Phase 4: Coverage
make coverage-docs
```

### GitHub CLI 操作

```bash
# 手动触发新 workflow
gh workflow run ci-v2.yml

# 查看运行状态
gh run list --workflow=ci-v2.yml

# 查看详细
gh run view <run-id>

# 下载 artifact
gh run download <run-id> -n nexus-test-archive
```

## 风险矩阵

| 风险 | 影响 | 概率 | 缓解措施 |
| ------ | ------ | ------ | --------- |
| nextest archive 不兼容某些测试 | 高 | 低 | Task 1.1 提前验证；回退到独立 Job |
| Docker image artifact 太大 | 中 | 低 | zstd 压缩；设置 retention-days=1 |
| 合并 Job 掩盖测试失败位置 | 中 | 中 | nextest 输出精确到测试名，可追踪 |
| 分支保护规则更新遗漏 | 高 | 低 | Task 3.3 清单验证 |
| 旧 workflow 清理不完全 | 低 | 低 | Task 4.1 清单验证 |
| coverage Job 缓存与 build 冲突 | 中 | 低 | 使用不同 cache key |
