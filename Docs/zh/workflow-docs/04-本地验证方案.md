# Nexus-Node 本地验证方案

> 日期：2026-03-31
> 适用阶段：v0.1.x 单人开发阶段（已移除 GitHub Actions）
> 前提：已执行 `make setup` 安装开发工具

---

## 1. 验证分级

本地验证按频率和场景分为四级，从快到慢：

```text
Level 0  编码中随手跑        ~10s    make check
Level 1  提交前             ~3min   make pre-push-fast
Level 2  推送/里程碑前       ~12min  make pre-push
Level 3  发版/重大变更前     ~25min  make pre-push + make devnet-full + make coverage
```

---

## 2. 各级命令详解

### Level 0：编码中（改了就跑）

```bash
make check          # cargo check --workspace，类型检查，不生成二进制，~10s
```

用途：写完一段逻辑后快速确认编译通过。等价于 IDE 的保存后自动检查。

### Level 1：提交前（快速门禁）

```bash
make pre-push-fast
```

展开等价于：

```bash
cargo fmt --all -- --check       # 格式检查
cargo clippy --workspace --all-targets -- -D warnings  # 静态分析
make verify-test-fixtures        # .mv 字节码完整性
cargo test                       # 单元测试 + 集成测试
```

**适用场景**：日常 commit，确保代码质量基线。

### Level 2：推送前（完整门禁）

```bash
make pre-push
```

展开等价于：

```bash
make lint                        # fmt-check + clippy
make security                    # cargo-audit + cargo-deny
make verify-test-fixtures        # 字节码完整性
cargo test                       # 全量测试
cargo test -p nexus-execution --features move-vm  # Move VM 测试
cargo test -p nexus-crypto -- --ignored --test-threads=1  # Crypto KAT
```

**适用场景**：推送到远程、合入 main、或完成一个功能模块时。

### Level 3：发版前（全面验证）

```bash
# 代码质量 + 安全 + 测试
make pre-push

# Docker 集群冒烟（构建镜像 → 7 节点 devnet → 健康检查 → 合约部署 → 清理）
make devnet-full

# 覆盖率报告（生成 LCOV + HTML + 记分卡 + 双语报告）
make coverage-docs
```

**适用场景**：打 tag 前、里程碑发布前、重大重构完成后。

## 3. 常用命令速查

### 构建

| 命令 | 用途 | 耗时 |
| ------ | ------ | ------ |
| `make check` | 类型检查（不生成二进制） | ~10s |
| `make build` | workspace 全量 release 构建 | ~12min |
| `make release` | 同 build（别名） | ~12min |

### 代码质量

| 命令 | 用途 | 耗时 |
| ------ | ------ | ------ |
| `make fmt` | 自动格式化 | ~2s |
| `make fmt-check` | 检查格式（不修改） | ~2s |
| `make clippy` | 静态分析（-D warnings） | ~3min |
| `make machete` | 检查未使用依赖 | ~5s |
| `make lint` | fmt-check + clippy | ~3min |

### 测试

| 命令 | 用途 | 耗时 |
| ------ | ------ | ------ |
| `make test` | cargo test（单线程，默认 features） | ~8min |
| `make nextest` | cargo-nextest 并行测试 | ~5min |
| `make test-vm` | Move VM 集成测试 | ~3min |
| `make test-kat` | 密码学 KAT 向量 | ~2min |
| `make doctest` | 文档测试 | ~1min |
| `make test-all` | nextest + doctest + KAT | ~8min |

### 安全

| 命令 | 用途 | 耗时 |
| ------ | ------ | ------ |
| `make audit` | CVE 漏洞扫描 | ~30s |
| `make deny` | 许可证 + 依赖来源 | ~20s |
| `make security` | audit + deny | ~1min |

### 覆盖率

| 命令 | 用途 | 耗时 |
| ------ | ------ | ------ |
| `make coverage` | LCOV 报告 | ~12min |
| `make coverage-html` | HTML 可视化报告 | ~12min |
| `make coverage-docs` | 全套报告（LCOV + HTML + 记分卡 + 双语 Docs） | ~15min |

### Docker Devnet

| 命令 | 用途 | 耗时 |
| ------ | ------ | ------ |
| `make devnet-up` | 构建镜像 + 生成配置 + 启动 7 节点 | ~8min |
| `make devnet-smoke` | 健康检查（需先 devnet-up） | ~1min |
| `make devnet-contracts` | 合约部署冒烟（需先 devnet-up） | ~2min |
| `make devnet-full` | up → smoke → contracts → down 全生命周期 | ~12min |
| `make devnet-down` | 停止容器 | ~5s |
| `make devnet-clean` | 停止 + 删除所有 devnet 状态 | ~5s |
| `make devnet-status` | 检查各节点健康状态 | ~2s |
| `make devnet-logs` | 实时跟踪日志 | 交互式 |
| `make devnet-tail N=2` | 跟踪指定节点日志 | 交互式 |
| `make devnet-bench` | TPS/延迟基准测试（需先 devnet-up） | ~5min |
| `make devnet-cold-bench` | 冷启动全流程基准 | ~15min |

### 字节码管理

| 命令 | 用途 |
| ------ | ------ |
| `make verify-test-fixtures` | 检查 .mv 文件包含正确的 dev-address |
| `make compile-test-fixtures` | 用 dev-address 重新编译所有 .mv |

## 4. 辅助脚本说明

`scripts/` 目录下的脚本按用途分类：

### 日常开发可用

| 脚本 | 用途 | 调用方式 |
| ------ | ------ | --------- |
| `setup-devnet.sh` | 生成验证者密钥和配置 | 通过 `make devnet-setup` |
| `generate-compose.sh` | 生成 docker-compose.yml | 通过 `make devnet-setup` |
| `smoke-test.sh` | devnet 健康检查 | 通过 `make devnet-smoke` |
| `contract-smoke-test.sh` | 合约部署/调用/查询 | 通过 `make devnet-contracts` |
| `config-doc-drift-check.sh` | 检查代码默认值与文档是否一致 | `zsh scripts/config-doc-drift-check.sh` |
| `validate-startup.sh` | 节点启动验证 | `bash scripts/validate-startup.sh` |
| `check-env.sh` | 环境依赖检查 | `bash scripts/check-env.sh` |

### 压力/安全测试

| 脚本 | 用途 | 说明 |
| ------ | ------ | ------ |
| `backpressure-test.sh` | 背压测试 | 需要运行中的 devnet |
| `soak-test.sh` | 浸泡测试 | 长时间运行，需要 devnet |
| `abuse-replay.sh` | 攻击重放测试 | 需要 devnet |
| `devnet-cold-bench.sh` | 冷启动基准 | 通过 `make devnet-cold-bench` |

### 报告生成

| 脚本 | 用途 | 说明 |
| ------ | ------ | ------ |
| `generate_coverage_scorecard.py` | 覆盖率记分卡 | 通过 `make coverage-docs` |
| `write_coverage_ci_summary.py` | CI summary 输出 | 通过 `make coverage-docs`（本地生成 md 文件） |

### 发版专用（当前阶段无需使用）

| 脚本 | 用途 |
| ------ | ------ |
| `release-go-nogo.sh` | 自动化发版检查清单（v0.2+ 恢复 CI 时启用） |
| `test-query.sh` | RPC 查询验证 |

## 5. 推荐日常工作流

```text
编码 → make check → 编码 → make check → ...
                                  ↓
                          功能完成，准备提交
                                  ↓
                          make pre-push-fast
                                  ↓
                              git commit
                                  ↓
                          功能分支完成，准备合入
                                  ↓
                          make pre-push
                                  ↓
                    涉及网络/Docker/合约相关修改？
                          ↓              ↓
                         是              否
                          ↓              ↓
                   make devnet-full    直接推送
                          ↓
                       git push
```

### 里程碑检查清单

```bash
# 1. 完整门禁
make pre-push

# 2. Docker 集群冒烟
make devnet-full

# 3. 覆盖率（可选，耗时较长）
make coverage-docs

# 4. 配置漂移检查
zsh scripts/config-doc-drift-check.sh

# 5. 查看覆盖率报告
open target/llvm-cov/html/index.html
```

## 6. 首次环境搭建

```bash
# 1. 安装 Rust 工具链（如未安装）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
rustup toolchain install 1.85.0
rustup default 1.85.0
rustup component add rustfmt clippy llvm-tools-preview

# 2. 安装开发工具
make setup

# 3. 验证环境
bash scripts/check-env.sh

# 4. 首次全量构建 + 测试
make build && make test-all

# 5. Docker 环境验证（需要 Docker Desktop/engine）
make devnet-full
```

## 7. 与未来 CI 的对应关系

当 v0.2+ 恢复 CI 时，本地命令与 CI Job 的映射关系：

| 本地命令 | 未来 CI Job |
| --------- | ------------ |
| `make fmt-check` | Pre-flight |
| `make clippy` | Build（合入编译） |
| `make security` | Pre-flight |
| `make test-all` | Core Tests |
| `make test-vm` | VM Tests |
| `make coverage-docs` | Coverage |
| `make devnet-full` | Smoke Tests |
| `make pre-push` | ≈ Phase 0 + Phase 1 + Phase 2 |

恢复 CI 时参照 `02-CI流程设计方案.md` 和 `ci-v2.yml` 实施即可。
