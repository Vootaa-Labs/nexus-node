# Nexus-Node CI 流程设计方案

> 版本：v2.0 草案
> 日期：2026-03-31
> 设计原则：**编译一次，测试多次，镜像复用，按需分层**

---

## 1. 设计原理

### 第一性原理

Rust 项目 CI 的根本瓶颈是 **编译**。nexus-node workspace 包含 15 个 crate + FFI 密码学依赖，冷编译 12-18 分钟。当前架构让每个 Job 独立编译，违反了最基本的工程原则——**消除冗余**。

### 核心策略

```text
编译产物是不可变工件（artifact），一旦构建成功就应当被所有后续阶段复用，
而不是每个消费者各自重新生产。
```

这意味着：

1. **一个 Build Job 产出所有工件**——release 二进制 + test 存档 + Docker 镜像
2. **测试 Job 零编译**——下载 nextest archive 直接运行，无需 Rust 工具链
3. **Docker 镜像从二进制构建**——使用 `runtime-prebuilt` target，跳过容器内编译
4. **冒烟测试复用同一镜像**——不重复构建

## 3. 核心技术方案

### 3.1 cargo nextest archive — 零编译测试

`cargo nextest archive` 将编译好的测试二进制和必要的元数据打包为一个可移植的存档文件。运行时只需 `cargo-nextest` 二进制本身，**不需要 Rust 工具链**。

```bash
# Build Job 中创建测试存档
cargo nextest archive --workspace --archive-file nexus-test-archive.tar.zst

# Test Job 中运行（无需 rustc/cargo）
cargo nextest run --archive-file nexus-test-archive.tar.zst --profile ci

# 运行特定测试子集（用于各专项门禁）
cargo nextest run --archive-file nexus-test-archive.tar.zst \
  -E 'test(forged_vote_should_be_rejected) | test(forged_transaction_should_be_rejected_at_gossip_ingress)'
```

**关键优势：**

- 测试 Job 无需 Rust 工具链（~2-3 min 节省/Job）
- 测试 Job 无需恢复编译缓存（~2-4 min 节省/Job）
- 每个测试 Job 仅 ~30s 初始化开销（下载 artifact + 解压）

### 3.2 Docker runtime-prebuilt — 跳过容器内编译

现有 Dockerfile 已设计好 `runtime-prebuilt` target：

```bash
# 使用 host 编译的二进制直接打包
cargo build --release -p nexus-node -p nexus-keygen -p nexus-genesis -p nexus-wallet
docker build --target runtime-prebuilt -t nexus-node .
```

在 Build Job 中一次编译、一次打包，产出的镜像供所有冒烟测试使用。

### 3.3 Docker 镜像传递方案

方案 A: **docker save/load（推荐用于 CI，简单可靠）**

```yaml
# Build Job
- run: docker save nexus-node:ci-$SHA | zstd -1 > nexus-image.tar.zst
- uses: actions/upload-artifact@v5
  with:
    name: nexus-docker-image
    path: nexus-image.tar.zst

# Smoke Job
- uses: actions/download-artifact@v5
  with:
    name: nexus-docker-image
- run: zstd -d nexus-image.tar.zst | docker load
```

方案 B: **GHCR 临时 tag（适用于需要多 runner 的场景）**

```yaml
# Build Job
- run: |
    echo "${{ secrets.GITHUB_TOKEN }}" | docker login ghcr.io -u ${{ github.actor }} --password-stdin
    docker tag nexus-node:ci-$SHA ghcr.io/${{ github.repository }}/nexus-node:ci-$SHA
    docker push ghcr.io/${{ github.repository }}/nexus-node:ci-$SHA
```

**推荐方案 A**，因为：避免 GHCR 写入权限配置、无网络延迟问题、PR fork 安全。

### 3.4 Clippy 合入 Build Job

Clippy 本质上是编译过程中的静态分析，将其从独立 Job 合入 Build Job 可以避免额外一次完整编译：

```bash
# Build Job 中：先 clippy 检查，通过后继续 build
cargo clippy --workspace --all-targets -- -D warnings
cargo build --release ...
```

如果 clippy 失败，整个 Build Job 提前失败，后续步骤不会执行——这与原来 lint Job 的语义一致。

### 3.5 格式检查与安全审计 — Pre-flight

这两类检查**不需要编译**，可以在 Build Job 之前并行快速完成：

| 检查 | 工具 | 是否需要编译 | 耗时 |
| ------ | ----- | ------------ | ------ |
| `cargo fmt --check` | rustfmt | ❌ | ~30s |
| `cargo machete` | cargo-machete | ❌ (仅解析 Cargo.toml) | ~10s |
| `cargo audit` | cargo-audit | ❌ (仅检查 Cargo.lock) | ~30s |
| `cargo deny check` | cargo-deny | ❌ (仅检查元数据) | ~20s |
| `config-doc-drift-check.sh` | zsh 脚本 | ❌ | ~5s |
| debug feature 检查 | cargo tree | ⚠️ (需要解析依赖，但不编译) | ~30s |

**Pre-flight 总耗时约 2 分钟，可在 Build 开始前完成快速失败。**

### 4.1 Job 数量

| | 旧 | 新 |
| -- | --- | --- |
| ci.yml Job 总数 | 20 | 8-10 |

### 4.2 时间线对比

**旧架构（热缓存），关键路径：**

```text
lint (8min)    ─────┐
security (6min)─────┤
check (8min)   ─────┤
test (20min)   ─────┼→ test-vm (15min) ─┐
                    │  correctness (12min)├→ release-go-nogo (15min) → move-vm-smoke (25min)
                    │  recovery (10min)  ─┤
                    │  ... × 10 more ────┘
                    │
                    └→ coverage (25min)
                       crypto-kat (10min)
                       
关键路径: ~50-65 min
总计费分钟: ~200-300 min
```

**新架构，关键路径：**

```text
pre-flight (2min) ──→ build (18min) ──→ test-gates (10min, 并行) ──→ smoke (10min) ──→ go-nogo (1min)
                                    └→ coverage (18min, 并行)
                                    
关键路径: ~32-40 min
总计费分钟: ~60-80 min
```

## 5. 详细 Job 设计

### Phase 0: Pre-flight

#### Job: `preflight`

```yaml
jobs:
  preflight:
    name: Pre-flight
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v6.0.2
      
      - name: Install Rust toolchain (fmt only)
        uses: dtolnay/rust-toolchain@...
        with:
          toolchain: "1.85.0"
          components: rustfmt
      
      # 格式检查（无需编译）
      - name: Check formatting
        run: cargo fmt --all -- --check
      
      # 安全审计（无需编译）
      - name: Install security tools
        run: |
          curl -sL https://github.com/rustsec/rustsec/releases/latest/download/cargo-audit-x86_64-unknown-linux-gnu.tgz | tar xzf - -C /usr/local/bin
          curl -sL https://github.com/EmbarkStudios/cargo-deny/releases/download/0.18.3/cargo-deny-0.18.3-x86_64-unknown-linux-gnu.tar.gz | tar xzf - --strip-components=1 -C /usr/local/bin
      
      - name: Cargo audit
        run: cargo audit --ignore RUSTSEC-2025-0009 --ignore RUSTSEC-2026-0049 --ignore RUSTSEC-2026-0009
      
      - name: Cargo deny
        run: cargo deny check licenses bans sources
      
      # 配置漂移检查
      - name: Config-doc drift check
        run: |
          sudo apt-get update && sudo apt-get install -y zsh
          zsh scripts/config-doc-drift-check.sh
      
      # Debug feature 检查
      - name: Check no debug features
        run: |
          if cargo tree --workspace --edges features 2>/dev/null | grep -q 'feature "debug"'; then
            echo "::error::Debug features detected in dependency tree"
            exit 1
          fi
```

### Phase 1: Build

#### Job: `build`

```yaml
  build:
    name: Build
    runs-on: ubuntu-latest
    timeout-minutes: 25
    needs: [preflight]
    steps:
      - uses: actions/checkout@v6.0.2
      
      - name: Install Rust toolchain
        uses: dtolnay/rust-toolchain@...
        with:
          toolchain: "1.85.0"
          components: clippy
      
      - uses: Swatinem/rust-cache@...
        with:
          cache-on-failure: true
          # 使用精确 key 避免缓存冲突
          key: unified-build
      
      - name: Install system dependencies
        run: sudo apt-get update && sudo apt-get install -y libclang-dev cmake clang
      
      # Clippy 检查（合入编译流程，避免独立编译）
      - name: Clippy
        run: cargo clippy --workspace --all-targets -- -D warnings
      
      # 安装 nextest（使用预编译二进制，不通过 cargo install）
      - name: Install cargo-nextest
        run: |
          curl -LsSf https://get.nexte.st/latest/linux | tar zxf - -C /usr/local/bin
      
      # 创建测试存档（默认 features）
      - name: Create test archive
        run: cargo nextest archive --workspace --archive-file nexus-test-archive.tar.zst
      
      # 创建 VM 测试存档（move-vm feature）
      - name: Create VM test archive
        run: |
          cargo nextest archive \
            -p nexus-execution -p nexus-wallet -p nexus-test-utils \
            --features move-vm \
            --archive-file nexus-vm-test-archive.tar.zst
      
      # Release 编译（用于 Docker 镜像和冒烟测试）
      - name: Build release binaries
        run: cargo build --release -p nexus-node -p nexus-keygen -p nexus-genesis -p nexus-wallet
      
      # 构建 Docker 镜像
      - name: Build Docker image
        run: docker build --target runtime-prebuilt -t nexus-node:ci-${{ github.sha }} .
      
      # 保存 Docker 镜像
      - name: Save Docker image
        run: docker save nexus-node:ci-${{ github.sha }} | zstd -1 > nexus-image.tar.zst
      
      # 上传所有工件
      - name: Upload test archive
        uses: actions/upload-artifact@v5
        with:
          name: nexus-test-archive
          path: nexus-test-archive.tar.zst
          retention-days: 1
      
      - name: Upload VM test archive
        uses: actions/upload-artifact@v5
        with:
          name: nexus-vm-test-archive
          path: nexus-vm-test-archive.tar.zst
          retention-days: 1
      
      - name: Upload Docker image
        uses: actions/upload-artifact@v5
        with:
          name: nexus-docker-image
          path: nexus-image.tar.zst
          retention-days: 1
      
      - name: Upload release binaries
        uses: actions/upload-artifact@v5
        with:
          name: nexus-release-binaries
          path: |
            target/release/nexus-node
            target/release/nexus-keygen
            target/release/nexus-genesis
            target/release/nexus-wallet
          retention-days: 1
```

### Phase 2: Test Gates

所有测试 Job 使用 **相同的模板**，只是测试过滤表达式不同：

```yaml
  # 模板：零编译测试 Job
  core-tests:
    name: Core Tests
    runs-on: ubuntu-latest
    timeout-minutes: 20
    needs: [build]
    steps:
      - uses: actions/checkout@v6.0.2  # 仅需 source（doc-tests 场景）
      
      - name: Install cargo-nextest
        run: curl -LsSf https://get.nexte.st/latest/linux | tar zxf - -C /usr/local/bin
      
      - name: Download test archive
        uses: actions/download-artifact@v5
        with:
          name: nexus-test-archive
      
      # 运行核心测试集
      - name: Run core tests
        run: cargo nextest run --archive-file nexus-test-archive.tar.zst --profile ci
      
      # Crypto KAT（known-answer tests）
      - name: Crypto KAT
        run: |
          cargo nextest run --archive-file nexus-test-archive.tar.zst \
            -p nexus-crypto -E 'test(/ignored/)' -- --ignored --test-threads=1
```

#### 测试 Job 合并编排

将原来 14+ 个独立测试 Job 合并为 6 个逻辑组：

| 新 Job | 合并自 | 测试内容 |
| -------- | ------- | --------- |
| **core-tests** | test, crypto-kat, check | nextest 全量 + KAT + doc-tests |
| **safety-tests** | correctness-negative, recovery, cold-restart | 安全负面测试 Phase A-F + 恢复测试 |
| **integration-tests** | startup-readiness, epoch-reconfig, proof-surface | 启动/Epoch/证明完整性 |
| **vm-tests** | test-vm, (move-vm-smoke 的测试部分) | Move VM 集成测试 |
| **formal-verify** | formal-verification, stake-weighted-quorum | 属性测试 + 形式验证 + 权益加权验证 |
| **capacity-gas** | capacity-curves, gas-calibration | 容量/Gas 校准 |

### Phase 3: Smoke Tests

```yaml
  smoke-tests:
    name: Devnet & Contract Smoke
    runs-on: ubuntu-latest
    timeout-minutes: 20
    needs: [build, core-tests, safety-tests, vm-tests]
    steps:
      - uses: actions/checkout@v6.0.2
      
      - name: Download Docker image
        uses: actions/download-artifact@v5
        with:
          name: nexus-docker-image
      
      - name: Load Docker image
        run: zstd -d nexus-image.tar.zst | docker load
      
      - name: Download release binaries (for devnet setup tools)
        uses: actions/download-artifact@v5
        with:
          name: nexus-release-binaries
          path: target/release
      
      - name: Make binaries executable
        run: chmod +x target/release/nexus-*
      
      # 引导 4 节点开发网
      - name: Bootstrap devnet
        run: |
          export PATH="$PWD/target/release:$PATH"
          ./scripts/setup-devnet.sh -n 4 -o devnet-ci -f
          ./scripts/generate-compose.sh -n 4 -o docker-compose-ci.yml -d devnet-ci
      
      - name: Start cluster
        run: docker compose -f docker-compose-ci.yml up -d
      
      - name: Wait for readiness
        run: |
          for i in 0 1 2 3; do
            PORT=$((8080 + i))
            ELAPSED=0
            while [ "$ELAPSED" -lt 180 ]; do
              HTTP_CODE=$(curl -s -o /dev/null -w '%{http_code}' "http://localhost:${PORT}/ready" 2>/dev/null || echo "000")
              [ "$HTTP_CODE" = "200" ] && break
              sleep 3; ELAPSED=$((ELAPSED + 3))
            done
            [ "$ELAPSED" -ge 180 ] && { echo "::error::node-$i not ready"; exit 1; }
          done
      
      - name: Verify consensus advancing
        run: ./scripts/smoke-test.sh
      
      - name: Contract smoke test
        run: ./scripts/contract-smoke-test.sh -k ./devnet-ci/validator-0/keys
      
      - name: Collect logs on failure
        if: failure()
        run: docker compose -f docker-compose-ci.yml logs --no-color > devnet-logs.txt
      
      - name: Upload logs
        if: failure()
        uses: actions/upload-artifact@v5
        with: { name: smoke-logs, path: devnet-logs.txt }
      
      - name: Teardown
        if: always()
        run: docker compose -f docker-compose-ci.yml down -v
```

## 6. 独立工作流保留策略

| 工作流 | 处理方案 |
| -------- | --------- |
| `ci.yml` | **重构**为上述新架构 |

## 7. 关于 Coverage 的特殊处理

Coverage 需要 **LLVM 插桩编译**，与常规编译产物不兼容，因此需要独立编译。这是 **唯一合理的第二次编译**。

优化策略：

1. Coverage 与测试门禁并行运行（都 depends on build，但 coverage 不等测试完成）
2. 使用独立缓存 key（`coverage`），避免与 build 缓存冲突
3. 如果 CI 预算紧张，可以考虑将 coverage 改为仅 main 分支运行

## 8. 迁移风险与应对

| 风险 | 概率 | 应对策略 |
| ------ | ------ | --------- |
| nextest archive 不支持某些测试模式 | 中 | `--ignored` 测试和 doc-tests 需特殊处理 |
| Docker image artifact 过大 | 低 | 使用 zstd 压缩，runtime-prebuilt ~50-80MB |
| artifact retention 导致存储膨胀 | 低 | 设置 `retention-days: 1` |
| 测试隔离性降低（合并 Job） | 低 | 保持 nextest 过滤表达式精确匹配 |
| doc-tests 需要 rustc | 中 | core-tests Job 额外安装 Rust toolchain |

### doc-tests 特殊处理

`cargo nextest` 不支持 doc-tests（这是已知限制）。解决方案：

- core-tests Job 中额外安装 Rust 工具链
- 或者，将 doc-tests 合入 build Job（编译时顺带运行）

**推荐方案：在 Build Job 中运行 doc-tests**，因为编译产物已在内存中，零额外编译开销。

## 9. 新增 Dockerfile.ci（可选增强）

如果团队需要更彻底的 Docker 化构建（所有编译在容器内完成），可以创建一个专用的 CI 构建镜像：

```dockerfile
# Dockerfile.ci — CI 构建镜像
# 预装所有构建工具，作为 CI 基础镜像
FROM rust:1.85.0-bookworm AS ci-base

RUN apt-get update && apt-get install -y --no-install-recommends \
    cmake clang pkg-config libssl-dev libclang-dev && \
    rm -rf /var/lib/apt/lists/*

# 预装 CI 工具
RUN cargo install cargo-nextest --locked && \
    cargo install cargo-llvm-cov --version 0.6.21 --locked

# 缓存依赖层（同 Dockerfile builder stage 的依赖缓存技巧）
```

这个镜像可以发布到 GHCR，作为所有 CI Job 的基础镜像，进一步消除工具安装开销。但这需要额外的镜像维护流程，属于 **可选增强**。
