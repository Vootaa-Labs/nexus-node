# 版本历史摘要：v0.1.0 → v0.1.12

本文件提供各历史版本的功能摘要。当前版本的详细分析请参阅 v0.1.13 系列报告。

## 版本演进

| 版本 | 主题 | 核心交付物 |
|------|------|-----------|
| v0.1.0 | 创世 | 初始工作区结构、核心 crate 布局 |
| v0.1.1 | Devnet 基础 | P2P 网络 (libp2p+QUIC)、Narwhal DAG 共识、链头、RocksDB 存储、意图执行管线 |
| v0.1.2 | 测试网加固 | 安全边界、输入验证、RPC 限流、认证层 |
| v0.1.3 | 执行正确性 | Block-STM 并行执行、共识验证、状态同步协议 |
| v0.1.4 | 运行时真实性 | `/ready` 健康端点、创世启动序列、Epoch 基础设施、证明 RPC |
| v0.1.5 | 证据级别 | 混沌/浸泡测试、证明校准、运维 Runbook、形式化验证引导 |
| v0.1.6 | 持久化加固 | DAG 恢复、BatchStore 持久化、冷启动韧性 |
| v0.1.7 | 经济基础 | Token 精度 (18→9 位小数)、权益加权法定人数、MVCC 共享数据结构 |
| v0.1.8 | 状态承诺 | BLAKE3 Merkle 状态承诺、排除证明、状态根生产化 |
| v0.1.9 | 链上 Staking | Staking 合约、由规范状态驱动的委员会轮换 |
| v0.1.10 | 多分片执行 | 分片感知执行、跨分片交易协调 |
| v0.1.11 | 文档与 Agent Core | 文档债务清理、Agent Core MCP 生产化、形式化验证深化 |
| v0.1.12 | 运行时加固 | Move VM 边界加固、gas/payload 接口规范化、配置外部化、PQC (ML-DSA-65) 集成 |

## 成熟阶段

v0.1.0–v0.1.12 的演进经历四个阶段：

1. **基础设施** (v0.1.0–v0.1.1)：crate 布局、网络、共识、存储
2. **Devnet 连通** (v0.1.2–v0.1.5)：安全加固、运行时正确性、证据收集
3. **经济与状态加固** (v0.1.6–v0.1.9)：持久化、Token 经济模型、状态证明、Staking
4. **能力闭环** (v0.1.10–v0.1.12)：多分片执行、Agent Core、PQC 密码学、Move VM 加固

## 文档边界

- **本文件** 覆盖 v0.1.0 至 v0.1.12 的历史版本，是摘要记录，非活跃规范。
- **当前版本文档**（`*_v0.1.13.md` 报告、指南、运维手册）描述系统的当前状态。
- 各版本的历史路线图和审计报告归档于开发仓库的 `Docs_Dev/` 中。
