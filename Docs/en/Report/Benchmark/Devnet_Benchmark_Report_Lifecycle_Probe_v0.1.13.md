# Devnet Benchmark Report v0.1.13

This report is generated by `cargo run -p nexus-bench --bin devnet_bench --release -- ...`.
Numbers below are devnet and cluster-visibility proxies, not production chain claims.

## Configuration

- Environment: shared-host Docker devnet
- Host resource budget: 8 CPU cores, 28 GiB RAM
- Interpretation boundary: results must be read against the shared-host constraint above, not as isolated bare-metal capacity
- Nodes: http://127.0.0.1:8080, http://127.0.0.1:8081, http://127.0.0.1:8082, http://127.0.0.1:8083, http://127.0.0.1:8084, http://127.0.0.1:8085, http://127.0.0.1:8086
- Worker sweep: [10, 12, 16]
- Transactions per worker: 10
- Transfer amount: 1000
- Shards: 2
- Confirmation timeout: 60000 ms
- Poll interval: 1000 ms

## Interpretation

- `local_tps`: confirmed transactions per second from submit start until receipts become visible on the submit node.
- `cluster_visibility_tps`: confirmed transactions per second until receipts become visible on every sampled node.
- `cluster_visibility_latency_ms`: a finality proxy based on cross-node receipt visibility, not a formal BFT finality proof.
