## Interpretation

- `local_tps`: confirmed transactions per second from submit start until receipts become visible on the submit node.
- `cluster_visibility_tps`: confirmed transactions per second until receipts become visible on every sampled node.
- `cluster_visibility_latency_ms`: a finality proxy based on cross-node receipt visibility, not a formal BFT finality proof.
